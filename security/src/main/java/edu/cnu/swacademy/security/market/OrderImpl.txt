6.1 주문 접수 - POST /api/v1/order
사용자가 특정 종목에 대해 매수/매도를 합니다.
지갑이 정지(is_blocked = true) 상태이면 주문 접수는 불가합니다.
주문 가격은 호가 단위 및 상/하한가 내에서만 유효합니다.
요청 정보를 토대로 주문(Order)을 생성하고, 현금/종목 지갑을 갱신합니다.

매수 주문인 경우
주문 생성 시 미체결 수량을 요청 수량으로 초기화합니다.
사용자 현금 지갑의 매수 주문으로 묶인 금액에 요청한 호가를 더합니다.
매도 주문인 경우
주문 생성 시 미체결 수량을 요청 수량으로 초기화합니다.
사용자 종목 지갑의 매도 수량에 요청한 수량을 더합니다.
주문 처리를 하기 위해 거래소 서버 주문 처리 API를 호출합니다.
생성된 주문은 거래소의 거래소에 전달되어 즉시 체결되거나, 체결 불가 시에는 오더북(orderbook)에 기록됩니다.

응답 결과 중 주문 접수 결과(match_result)에 따른 후속 처리
Unmatched(미체결)일 경우, 후속 조치가 필요하지 않습니다.
Matched(체결 완료)일 경우 현금(cash_wallet)/종목(stock_wallet) 지갑, 체결 내역(match), 장 상태(market_status) 업데이트가 필요합니다.
maker가 매수자일 경우
maker의 현금 지갑의 매수 주문으로 묶인 금액을 체결된 가격만큼 차감합니다.
갱신된 내역을 기록하기 위해 현금 지갑 내역을 생성합니다.
taker(매도자) 종목 지갑의 매도 수량을 체결된 수량만큼 차감합니다.
갱신된 내역을 기록하기 위해 종목 지갑 내역을 생성합니다.
체결 내역을 생성합니다.
장 상태(고가, 저가, 거래량, 거래대금 등) 업데이트가 필요한 컬럼만 갱신합니다.
maker가 매도 주문자일 경우
maker의 종목 지갑에서 매도 수량을 체결된 수량만큼 차감합니다.
갱신된 내역을 기록하기 위해 종목 지갑 내역을 생성합니다.
taker(매수자) 현금 지갑의 매수 주문으로 묶인 금액을 체결된 가격만큼 차감합니다.
체결 내역을 생성합니다.
갱신된 내역을 기록하기 위해 현금 지갑 내역을 생성합니다.
장 상태(고가, 저가, 거래량, 거래대금 등) 업데이트가 필요한 컬럼만 갱신합니다.
Rejected(거절)일 경우
매수 주문인 경우
거절된 수량 만큼 거래소 서버에 요청 전 생성한 주문(Order)의 미체결 수량을 차감하고, 취소 수량을 증가시킵니다.
거래소 서버에 요청 전 갱신한 현금지갑의 매수 주문으로 묶인 금액을 거절된 금액만큼 차감합니다.
갱신된 내역을 기록하기 위해 현금 지갑 내역을 생성합니다.
매도 주문인 경우
거절된 수량 만큼 이전에 생성한 주문(Order)의 미체결 수량을 감소시키고, 취소 수량을 증가시킵니다.
거래소 서버에 요청 전 갱신한 종목지갑의 매도 수량을 거절된 수량만큼 차감합니다.
갱신된 내역을 기록하기 위해 종목 지갑 내역을 생성합니다.
본 API를 호출한 사용자에게 거절 사유를 응답합니다.
요청
구분	필드	필수	타입	제약 조건	설명
Header	Authorization	Y	string	Bearer <token>	-
Body	stock_id	Y	int	>0	종목 ID
Body	side	Y	string(enum)	BUY/SELL	매수/매도
Body	price	Y	int	>0,	호가
Body	quantity	Y	int	>0,	수량
발생 가능한 모든 응답코드
코드	설명
200	주문 생성
400	유효성 오류
403	자산 부족
404	종목 없음
412	지갑 정지
416	규칙 위반(상·하한/틱)
500	서버 오류
성공 응답
필드	설명	필수	타입	제약 조건
reason	거절 사유	N (match_result가 Rejected인 경우에만 존재)	string	<=100
